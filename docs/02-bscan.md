# 超声成像算法实战——B扫与一维线性扫查

上一篇把一条A扫从头到尾走了一遍：模拟回波、滤波、包络、TGC、门控。一条A扫只说明探头停在一个点上看到了什么。这一篇让它动起来——探头沿一条线走，把每个位置采到的A扫按位置排成一幅图，这就是B扫。

仿真对象也换了个花样：一个四阶台阶模型，四段厚度分别是6.25、12.5、18.75、25mm。台阶尺寸照着 [Evident 的 2212M 厚度校准试块](https://ims.evidentscientific.com/zh/probes/single-and-dual-element/test-blocks)取的。这里全程还是仿真，没有用探头试块采集真实数据，就是拿这几个尺寸在代码里建了个模型来模拟回波——讲一下原理。

## **B扫**

B扫（Brightness scan）是沿一条扫查线展开的“深度—位置”截面，相当于给材料拍了一张侧视图：一个轴是探头沿表面移动的位置，一般称为扫查轴，另一个轴是深度，像素多亮代表回波多强。

![B扫原理](https://cdn.jsdelivr.net/gh/Achilles96925/ultrasonic-ndt-lab@main/algorithms/02_bscan/bscan_principle.png)

探头贴着模型表面沿x方向走，每隔一个固定间距收发一次超声脉冲，接收回波产生一条A扫。

## **A扫拼合**

B扫其实没什么新的采集原理，就是把A扫按位置排起来。每个位置采一条A扫，按x坐标一列列并排放好，就得到一个二维矩阵：行对应时间（也就是深度），列对应扫查位置。

![A扫拼合](https://cdn.jsdelivr.net/gh/Achilles96925/ultrasonic-ndt-lab@main/algorithms/02_bscan/bscan_formation.png)

左边用三维形式画了50条处理后的A扫，每条曲线上都是上一篇最终形成的回波包络。把这些包络按位置紧紧排在一起、再把幅值映射成颜色，就是右边那幅平面B扫图像。

有个细节容易被忽略：组成一幅B扫的所有A扫，必须用**完全一样**的参数去处理（同样的滤波带宽、同样的包络算法），列和列之间的亮度才有可比性。不然有的列亮、有的列暗，看到的就不是真实的回波差异，而是处理参数的差异。

## **深度换算**

注意到B扫图像的纵轴标着毫米，可采到的原始数据里根本没有深度这个量，A扫中横轴是用的时间表示。那这里的深度依然是使用这个公式：
```
深度 = 声速 × TOF / 2
```

声速就根据试块的材质来获取，碳钢里的纵波声速大约5900m/s。实际检测有可能会有楔块，声束需要经过楔块内部和工件，还有可能不是垂直入射，计算规则就复杂起来了，这里先不讲。

## **RF vs 包络**

上面拼合B扫用的是**包络A扫**，没有选择原始的RF A扫，可以直接用图像对比来说明原因。

RF波形使用了黑白色带，每个回波都成了正负相间的黑白竖纹，和斑马一样，很难看出具体的信号强度：

![RF B扫](https://cdn.jsdelivr.net/gh/Achilles96925/ultrasonic-ndt-lab@main/algorithms/02_bscan/bscan_workbench_rf.png)

取了包络之后就完全不一样了。每个台阶的信号都会形成一团连续的色块，看起来一目了然：

![包络B扫](https://cdn.jsdelivr.net/gh/Achilles96925/ultrasonic-ndt-lab@main/algorithms/02_bscan/bscan_workbench_envelope.png)

拿25mm那个台阶来看，往下约25mm处那条水平亮线就是底面回波；四个台阶厚度不同，底波落在四个不同深度上，一张图就看出底面（也就是厚度）沿x是怎么变的。

## **局限**

B扫说到底只是一条扫查轴上的截面，扫查轴的位置限制了超声脉冲回波覆盖的部分，就好比医院里的CT，如果机器不移动，只会有一个截面的图像，医生无法准确分析病灶，所以只有B扫，也不足以准备分析缺陷信息。
